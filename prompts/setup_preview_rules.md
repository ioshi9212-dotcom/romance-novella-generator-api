# Setup Preview Rules

Перед первой сценой пользователь должен подтвердить основу новеллы.

Поток:

1. Пользователь заполняет стартовую анкету одним сообщением.
2. GPT создаёт bootstrap JSON.
3. GPT НЕ сохраняет его как активную игру.
4. GPT вызывает `createBootstrapPreview`.
5. Пользователю показывается preview: героиня, знакомые, работа/роль, стартовая локация, отношения, story_plan.
6. Только после явного подтверждения GPT вызывает `confirmBootstrapPreview`.
7. Только после подтверждения можно писать первую сцену.

Preview должен быть читабельным, без JSON, без скрытых future_locks и без технической простыни.

В preview обязательно показать:

- имя/ID главного персонажа;
- кто она/он, возраст, роль или работа;
- внешность коротко;
- характер, цель, привычки;
- кого знает на старте;
- где начинается история;
- главную завязку;
- цель новеллы/персонажа;
- главный конфликт;
- открытые сюжетные нити;
- стартовые отношения;
- 2 story-specific поля состояния.

Если пользователь просит правки, создать новый bootstrap JSON и снова показать preview. Не сохранять state до подтверждения.
