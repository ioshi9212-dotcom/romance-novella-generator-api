# Knowledge Rules

State не равен знанию NPC. Персонаж может говорить и действовать только из того, что он знает, видел, слышал, прочитал, получил через сообщение или может предположить по видимым признакам.

## Источники знания

Допустимые источники:

- текущая сцена, если персонаж присутствует;
- его `knowledge[character_id].knows`;
- загруженная relationship history, если она доступна этому персонажу;
- прямой диалог;
- видимые физические следы;
- сообщение/звонок/документ, который он реально получил;
- чужой пересказ, если он был сыгран или сохранён.

## Запреты

NPC не знает:

- мысли игрока в скобках;
- скрытые поля future_locks;
- истинные роли будущих персонажей;
- факты из сцен, где его не было;
- то, что знает другой NPC, если ему не сообщили;
- то, что лежит в авторском описании, но не стало сценическим фактом.

## Knowledge patch

Любой `knowledge_patch` обязан иметь источник:

```json
{
  "character_id": "kai_mori",
  "add_knows": ["Akira Vale вернулась в город"],
  "reason": "увидел её на станции",
  "source_in_scene": "Kai Mori физически увидел Akira Vale в текущей сцене"
}
```

Если нет `source_in_scene`, patch должен быть отклонён.

## Wrong beliefs

Если персонаж делает вывод по неполным видимым данным, лучше сохранить это как wrong belief / assumption, а не как истину мира.

Пример:

```json
{
  "character_id": "noah_akiyama",
  "add_knows": [],
  "add_assumptions": ["думает, что Keira Hart приехала из-за бывшего"],
  "source_in_scene": "увидел её реакцию на имя Elias Mori"
}
```
