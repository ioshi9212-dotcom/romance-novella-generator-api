# Knowledge Rules

State не равен знанию NPC. Персонаж может говорить и действовать только из того, что он знает, видел, слышал, прочитал, получил через сообщение или может предположить по видимым признакам.

## Где хранится

В v8 знания сохраняются отдельно на каждого сгенерированного персонажа:

```txt
state/knowledge/<character_id>.json
```

`character_id` создаётся в bootstrap или при появлении нового важного NPC. В репозитории нет заранее готовых персонажей и заранее готовых knowledge-файлов.

## Главное

Сохраняй не только объективный факт, а субъективную память персонажа:

- что он видел;
- что он слышал;
- какую фразу запомнил;
- как он это понял;
- какой вывод сделал;
- где он может ошибаться;
- какие вопросы у него остались.

## Источники знания

Допустимые источники:

- текущая сцена, если персонаж присутствует;
- его `state/knowledge/<character_id>.json`;
- загруженная relationship history, если она доступна этому персонажу;
- прямой диалог;
- видимые физические следы;
- сообщение/звонок/документ, который он реально получил;
- чужой пересказ, если он был сыгран или сохранён.

## Запреты

NPC не знает:

- мысли игрока в скобках;
- скрытые поля future_locks;
- истинные роли будущих персонажей;
- факты из сцен, где его не было;
- то, что знает другой NPC, если ему не сообщили;
- то, что лежит в авторском описании, но не стало сценическим фактом.

## Knowledge patch

Любой `knowledge_patch` обязан иметь:

- `character_id`;
- `reason`;
- `source_in_scene`.

Предпочтительный формат:

```json
{
  "character_id": "created_character_id",
  "add_observations": [
    {
      "saw": "что персонаж увидел",
      "heard": "что персонаж услышал",
      "remembered_quote": "короткая фраза, если важно",
      "interpreted_as": "как он это понял",
      "emotional_marker": "что зацепило",
      "certainty": "low / medium / high"
    }
  ],
  "add_assumptions": [
    {
      "text": "какой вывод сделал персонаж",
      "based_on": "видимый источник",
      "may_be_wrong": true
    }
  ],
  "add_recent_memories": ["короткая память сцены"],
  "add_open_questions": ["что персонаж теперь хочет понять"],
  "reason": "почему это знание появилось",
  "source_in_scene": "конкретная реплика/действие/видимое событие"
}
```

Если нет `source_in_scene`, patch должен быть отклонён.

## Wrong beliefs

Если персонаж делает вывод по неполным видимым данным, сохраняй это как `assumption` или `wrong_beliefs`, а не как истину мира.
