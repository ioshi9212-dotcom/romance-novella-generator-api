# Character Creator Rules

Создавай персонажа коротко, но достаточно для поведения.

## Генератор, не готовый канон

В репозитории нет готовых персонажей. Каждый новый персонаж создаётся только внутри конкретной сессии и получает свой `character_id`. Неизвестный будущий человек до фактического входа существует только как seed без имени и внешности.

Нельзя использовать имена, id, связи или роли из старых новелл как дефолт.

## Один персонаж = одна анкета

Один значимый персонаж сохраняется как один файл:

```txt
characters/<character_id>.json
```

Никаких отдельных `main/character/knowledge/past`.

## Обязательные поля

- id
- name
- role
- age
- introduced
- known_to_player
- locked
- appearance
- personality
- goal
- past_short
- habits
- likes_in_people
- dislikes_in_people
- relationship_triggers
- connections
- source_seed_id, если scene contract выдал character_creation_request

## Качество

Персонаж должен иметь:

- цель;
- привычку / жест / узнаваемый ритм поведения;
- дефект;
- манеру речи;
- то, что ему нравится в людях;
- то, что его отталкивает;
- условия, при которых отношения с ним улучшаются/ухудшаются;
- связь хотя бы с одним другим персонажем, если он не случайный фоновый NPC.

## likes / dislikes

`likes_in_people` и `dislikes_in_people` нужны не для красоты. Они влияют на relationship patches.

Пример логики без готовых имён:

```txt
если персонаж ценит прямоту, честный ответ может повысить respect/trust;
если персонаж ненавидит уклончивость, уход от ответа может повысить tension;
если персонаж боится хаоса, импульсивное действие может снизить trust;
если персонаж уважает самоконтроль, сдержанная реакция может повысить respect.
```

## Нельзя

- делать всех красивыми одинаково;
- делать NPC функцией для терапии героя;
- делать всех одинаково удобными;
- менять locked-внешность/базовый характер после сохранения;
- вводить “загадочного красавца” без роли в истории;
- копировать персонажей из готовых историй.

## Naming

Персонажи должны иметь не русские и не славянские имена/фамилии. Поле `name` — только латиницей. Предпочитай западно-японскую/англо-японскую стилистику, но не используй готовые имена из документации как список по умолчанию.
