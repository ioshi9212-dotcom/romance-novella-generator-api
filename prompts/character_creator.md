# Character Creator Rules

Создавай персонажа коротко, но достаточно для поведения.

## Один персонаж = одна анкета

Никаких main/character/knowledge/past по отдельности.

## Обязательные поля

- id
- name
- role
- age
- introduced
- known_to_player
- locked
- appearance
- personality
- goal
- past_short
- habits
- likes_in_people
- dislikes_in_people
- connections

## Поля поведения

Каждый значимый персонаж должен иметь короткие поведенческие якоря:

```json
{
  "goal": "чего персонаж хочет сейчас или в этой арке",
  "habits": ["2-4 узнаваемые привычки/жеста/ритма поведения"],
  "likes_in_people": ["что повышает доверие/интерес/уважение"],
  "dislikes_in_people": ["что снижает доверие/интерес/уважение"],
  "relationship_triggers": {
    "improves_when": ["видимое поведение, которое может улучшить отношение"],
    "worsens_when": ["видимое поведение, которое может ухудшить отношение"]
  }
}
```

Это не анкета на три страницы. 2-4 пункта достаточно.

## Качество

Персонаж должен иметь:

- цель;
- привычку;
- дефект;
- манеру речи;
- то, что ему нравится в людях;
- то, что ему не нравится в людях;
- связь хотя бы с одним другим персонажем, если он не случайный NPC.

## Как это влияет на отношения

Если персонаж игрока ведёт себя так, что совпадает с `likes_in_people` / `relationship_triggers.improves_when`, отношения могут немного улучшаться.

Если персонаж игрока делает то, что попадает в `dislikes_in_people` / `relationship_triggers.worsens_when`, отношения могут ухудшаться.

Сдвиг отношений всё равно требует видимого источника в сцене и `source_in_scene` в relationship patch.

## Нельзя

- делать всех красивыми одинаково;
- делать всех удобными;
- делать NPC функцией для терапии героя;
- делать NPC философами по умолчанию;
- менять внешность после сохранения;
- вводить “загадочного красавца” без роли в истории.

## Naming

Персонажи должны иметь не русские и не славянские имена/фамилии. Поле `name` — только латиницей. Предпочитай западно-японские/англо-японские сочетания: `Akira Vale`, `Noah Akiyama`, `Mika Lawson`, `Ren Carter`, `Yuna Cross`.
