# GPT Actions Contract — v9.2

Custom GPT is the writer. Railway stores memory, validates JSON, builds scene prompts, and applies state updates.

## Required launch flow

1. User says `начнем` or gives too little setup.
2. GPT calls `getStartQuestionnaire`.
3. User answers the questionnaire in one message. A partial answer is valid; missing details are generated by GPT.
4. GPT calls `createSession` with the exact complete answer in `raw_start_text` and `mode: gpt_actions`.
5. Railway returns `bootstrap_prompt` and `session_id`; session status is `bootstrap_pending`.
6. GPT builds one complete `bootstrap_json` and calls `createBootstrapPreview` once. Unknown future people are short seeds, not full character cards.
7. Railway normalizes technical fields, rejects only broken structure or impossible references, saves `pending_bootstrap.json`, and returns a human-readable preview. Source-fidelity heuristics are diagnostics and never create a repair loop.
8. GPT reads all preview chunks, shows the complete preview, and waits.
9. If the user asks for edits, GPT sends one updated complete `bootstrap_json` to `createBootstrapPreview` again.
10. If the user confirms, GPT calls `confirmBootstrapPreview`.
11. Railway commits state files and session becomes `active`; GPT may then call `processTurn` with `(начать первую сцену)`.

## Hard rules

- Do not call `applyBootstrapResult` during normal Custom GPT launch flow. It is legacy/direct save.
- Call `createBootstrapPreview` with exactly one top-level field, `bootstrap_json`; do not spill bootstrap sections into Action kwargs.
- Do not call `processTurn` before `confirmBootstrapPreview` succeeds.
- Do not show raw bootstrap JSON to the user unless they explicitly ask.
- Show only the preview during setup review.
- Send `turn_id` and scene fields flat to `applyTurnResult`; do not wrap them in `scene_response` and do not send `rendered_text`.
- Railway builds and retains the visible scene. After normal turns, show only `message_to_user`.
- A preview response contains visible text only once. When chunked, concatenate `message_to_user` (chunk 0) with every later `getBootstrapPreviewChunk` result. After a lost/`ResponseTooLargeError` response, read `diagnostics.bootstrap.preview_transport` from `debugSessionDump` and fetch chunks starting at index 0; do not rebuild the preview.
- A repeated `applyTurnResult` for the same applied `turn_id` is idempotent and replays the saved response.
- If the Action response is lost, call `getLastScene`; never create a replacement turn.

## State layout after confirm

```txt
characters/<character_id>.json
state/knowledge/<character_id>.json
state/relationship_pairs/<a>__<b>.json
story_plan.json
current_state.json
```

## Bootstrap core must include

- characters
- story_plan
- current_state
- future_locks.hidden_character_seeds for any unknown future people

Railway derives or normalizes `protagonist`, `relationships`, `knowledge`, `npc_state`, `director_bible`, `continuity`, `scene_history`, and `turns`. A partial questionnaire and `Рандом` are valid; GPT invents omitted creative details in the same bootstrap instead of asking again.

## Runtime scene context

`processTurn` sends GPT the current frame plus compact full cards only for the player and currently relevant known characters. At most one eligible future seed is exposed as an optional `character_creation_request`. If that person causally enters the written scene, GPT returns one complete `known_core`/`known_support` card with the exact `source_seed_id`; Railway creates its knowledge, runtime and player relationship and consumes only that seed in the same atomic turn write. If no entrance occurs, no card is created and the seed remains available. Legacy `hidden_core` sessions keep their existing atomic reveal protocol.

`story_plan` must include `setting_summary`, `main_premise`, `protagonist_start`, `player_goal`, `central_conflict`, `central_question`, `opening_scene_intent`, `act_structure`, `character_arcs`, `relationship_focus`, `open_threads`, `forbidden_drift`, `current_story_position`, `status_slots`.
