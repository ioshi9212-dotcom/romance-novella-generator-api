# romance-novella-generator-api — hotfix v6

## Постоянное правило для будущих правок

Не добавлять новые runtime-patch/locks/rules слои без прямого запроса пользователя.

Правильный стиль фиксов:
- чинить существующие `app/*.py`, prompts/schemas/tests;
- делать ZIP на замену поверх репозитория;
- давать `DELETE_LIST.txt`, даже если удалять нечего;
- если нужна инструкция для Custom GPT — давать отдельный `CUSTOM_GPT_INSTRUCTION_PATCH_v*.txt`;
- не плодить новые runtime-файлы, patch-файлы и lock-файлы.

## Что чинит v6

v4/v5 уже чинили сохранение, turn_id, короткий dashboard и базовую агентность NPC.
v6 чинит стиль сцены:

- меньше атмосферного полотна;
- больше коротких beats;
- реплики внутри body, а не отдельным блоком `Диалог:`;
- формат реплик: `**Имя** — Реплика. *(ремарка)*`;
- не собирать все реплики списком в конце;
- NPC должен проявляться 2–4 раза за сцену, если он есть;
- body 900–1800 знаков, 8–20 коротких beats/абзацев;
- описание не больше половины body, если есть живые персонажи;
- сарказм/ирония редко: 1–2 коротких момента на сцену.
